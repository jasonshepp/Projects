Testing for Tolerance¶

Jason Sheppard

Research Question¶

What tolerance value produces the most realistic-looking graphs that align with our expectations of stable and unstable behavior?

Stable Orbit Definition¶

A 'Stable' orbit refers to an orbit where an electron completes a full orbit such that when $x = x_{initial}$ our difference, $|y-y_{initial}|$ is less than our chosen tolerance value.

Shape detection¶

We choose how to define the shape of an orbit by checking if the electron goes between both protons, below the lower, both or neither. This gives us the 4 orbit shapes of around 1 proton, racetrack around both, figure 8 or an instant exit respectively. We can see that later on, this initial assumtion falls apart as more shapes appear.

Configuration¶

1 Electron at $(x,y) = (0,2)$ and 2 Protons $(x,y) = (0,1),(0,-1)$

Scale¶

All values (mass of particles and G) have been scaled so we can use $x=a[m]$ and $v = a[m/s]$

n_orbit = 0
def diff_eqns(t, state):
    x = state[0]
    y = state[1]
    vx = state[2]
    vy = state[3]
    
    global n_orbit
    dxdt = vx
    dydt = vy
    
    dvxdt = 0
    dvydt = 0
    
    
    for (xf, yf) in fixed_points:
        
        r = np.sqrt((x-xf)**2 + (y-yf)**2)
        u_x = (x - xf) / r
        u_y = (y - yf) / r
        
        dvxdt -= ((G * m_p) / r**2 ) * u_x  #acceleration x
        dvydt -= ((G * m_p) / r**2 ) * u_y  #acceleration y
    
    

    
    return dxdt, dydt, dvxdt, dvydt

def shape_num(sol1, tolerance):
    # Initializing var
    between_bool = False
    below_bool = False
    stable = True

    # Did the electrons cross the y-axis between protons? 
    # Did it cross below the lower?
    crossings = sol1.y_events[0]
    for event in crossings:
        y = event[1]
        if 1 > y > -1:
            between_bool = True
        if y < -1:
            below_bool = True

    # Did the electron cross the y-axis above the upper proton while being within 'tolerance' of its initial position?
    directions = sol1.y_events[1]
    for event in directions:
        y = event[1]
        if abs(y - 2) > tolerance and y > 1:
            stable = False
            break
    # Did the electron only cross once? (Flew off screen)
    if len(sol1.t_events[1]) <= 1:
        stable = False




    # Returning values according to logic from above

    # Flew off screen 
    if len(sol1.t_events[2]) > 0 or len(sol1.t_events[3]) > 0 or len(sol1.t_events[4]) > 0 or len(sol1.t_events[5]) > 0:
        return 0


    if below_bool and between_bool and stable:
        # Figure 8 - stable
        return 1
    elif between_bool and stable:
        # Around 1 - stable
        return 2
    elif below_bool and stable:
        # Racetrack - stable
        return 3
    # No case of stable orbit without going between or below
    elif below_bool and between_bool and not stable:
        # Figure 8 - unstable
        return 4
    elif between_bool and not stable:
        # Around 1 - Unstable
        return 5
    elif below_bool and not stable:
        # Racetrack - Unstable
        return 6
    else:
        # Wont ever run, just a catch all else statement (unsure if these are needed in python)
        return 0

def shape_checker(sol1, tolerance):
    between_bool = False
    below_bool = False
    stable = True
    string = ''

    crossings = sol1.y_events[0]
    for event in crossings:
        y = event[1]
        if 1 > y > -1:
            between_bool = True
        if y < -1:
            below_bool = True

    directions = sol1.y_events[1]
    for event in directions:
        y = event[1]
        if abs(y - 2) > tolerance and y > 1:
            stable = False
            break
    if len(sol1.t_events[1]) <= 1:
        stable = False

    if len(sol1.t_events[2]) > 0 or len(sol1.t_events[3]) > 0 or len(sol1.t_events[4]) > 0 or len(sol1.t_events[5]) > 0:
        return 'Electron flew off screen (Unstable)'

    if below_bool and between_bool:
        string += 'Figure 8'
    elif below_bool:
        string += 'Race Track'
    elif between_bool:
        string += 'Orbit around one proton'

    if stable:
        return 'Stable ' + string
    else:
        return 'Unstable ' + string

Conclusion¶

Using all the graphs above we can get a better understanding of which tolerance makes the most sense.

As we can see from graph 1, when we have a low tolerance none of our orbits will ever be classified as stable. Looking at graphs 3 and 4, both of these are perfectly symmetrical throughout repetition and thus should be defined as stable. This means our tolerance value cant be 0.001 or 0.002.

Looking at the highest tolerance values in graph 1, we see that there are many more stable orbits. This is because as the tolerance increases, our definition of stable becomes less strict. Looking at graph 5 makes us realize this is not a good thing. This orbit is way to chaotic to be classified as stable and thus we can consider any high tolerance value (>=0.512) as too forgiving and unusable.

Looking at graph 6 we can see that although its much less chaotic than graph 5, it still deviates from a clean symmetrical orbit and over even more time would become chaotic. Using this last graph narrows our range of acceptable tolerances to 0.016 and 0.032. Mostly all orbits that are stable when our tolerance = 0.032 are considered as stable when our tolerance = 0.016. Picking the most strict tolerance value that still matches our graphical expectations leaves us with a tolerance of 0.016.

Thus our definition for stability is: A 'Stable' orbit refers to an orbit where an electron completes a full orbit such that when $x = x_{initial}$ our difference, $|y-y_{initial}|$ is less than 0.016[m]

Conclusion 2¶

From both heat maps above, the whiter the line, the longer the orbit will be stable for. We see a pattern of the whitest lines around the v0 values that were stable in graph 2. We can see that to the right and left of each of these lines are slightly darker lines that indicate that the orbit was stable for slightly less orbits. We can also assume that any white line would will run indefinitely.

One suprising thing about this graph is that our strongest point is not one of our starting shapes. It is represented by graph 7 (reverse teardrop at v0 = 17.19). If we were to look past this outlier (second graph) our graph would contain many more apparent shapes. In the second graph we can see more lighter lines appear, better representing the data set.

This second graph does a better job of represeting the data set by telling us that there are more v0 values that create semi-stable orbits (temporarly stable) and intersting shapes than originally thought, which is to be after the first conclusion/set of graphs.

The heatmap also shows us regions where we are more likely to find a stable orbit (right side) as compared to regions were there are very few stable orbits (left and middle).
